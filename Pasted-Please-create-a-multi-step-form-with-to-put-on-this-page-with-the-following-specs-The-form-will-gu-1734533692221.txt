Technical Specifications
Multi-Step Form

Use a step navigation system with Previous, Next, and Submit buttons.
Submit button is visible only in the last step.
Input Validation

Required fields must be validated with appropriate error messages.
UI Considerations

Clean, responsive UI using a consistent design system.
Placeholders and help text should improve user clarity.
Real-Time Revenue Fetching

The “Get Revenue Data” button should call an API to fetch real-time revenue data (e.g., nightly rate and occupancy).
Data Persistence

Store form state locally as users navigate between steps.
Ensure that no data is lost if users move back and forth between steps.
Goal of the Form
The goal is to allow users to analyze a property for investment purposes, including purchase details, financing terms, operating expenses, revenue performance, and growth assumptions.

This form is crucial for generating detailed property analysis reports that reflect investment metrics.
This is basically the starting point for a property analyser engine. The form input makes up the SaaS component of the product, but there is also an API-as-a-service which we will work on later.
