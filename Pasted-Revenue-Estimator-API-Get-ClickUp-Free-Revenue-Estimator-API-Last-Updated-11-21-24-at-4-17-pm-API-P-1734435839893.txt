API Documentation
Input to the API

Address or Latitude, Longitude

You can type the address, or give Latitude, Longitude of the location. For most accurate results, Latitude and Longitude is recommended

Bedroom Category

The Bedroom Category for which you want the result. If your bedroom category is studio, it will translate to 0 and if its Room it will translate to -1 in the input. For 1BR, its 1, for 2BR its 2, and so on and so forth

Currency

The currency for which you want the results. Please use standard short-form of the currency to get the results. So USD, CAD, AUD, etc.

Filters

pool: 0 or 1; 0 means pool=no, 1 means pool=yes

hottub: 0 or 1; 0 means hottub=no, 1 means hottub=yes

ratings: using this parameter, apply filter for ratings of properties you want to consider

review_counts: using this parameter, apply filter for review counts on properties

bathroom: using this parameter, filter for properties more than or less than certain bathroom count

max_guests: using this parameter, filter for properties more than or less than certain max_guest count



Output of the API

Revenue

Revenue50PercentileSum ➝ This is the estimated annual revenue, if you want to share with anyone one number, this would be it

RevenueMonthlyAvg ➝ This is average revenue per month. Achieved by dividing Revenue50PercentileSum by 12

Revenue25PercentileSum ➝ This is lower bound of the annual revenue number. This is an estimate for listings that are in 25th percentile bracket

Revenue75PercentileSum ➝ This is upper bound of the annual revenue number. This is an estimate for listings that are in 75th percentile bracket

Revenue90PercentileSum ➝ This is upper bound of the annual revenue number. This is an estimate for listings that are in 90th percentile bracket

ADR

ADR50PercentileAvg ➝ This is average over 12 months for listings with ADR in 50th percentile. This is also the average number that you want to use at high-level

ADR75PercentileAvg ➝ This is average over 12 months for listings with ADR in 75th percentile

ADR25PercentileAvg ➝ This is average over 12 months for listings with ADR in 75th percentile

ADR90PercentileAvg ➝ This is average over 12 months for listings with ADR in 90th percentile

Occupancy

AvgAdjustedOccupancy ➝ This is average occupancy for the year adjusted for owner blocks

Number of listings used

Number of listings used to arrive at the estimate. Its a mix of active and inactive listings.

Frequently Asked Questions (FAQs)


Calculation Methodologies

How is Annual Revenue Calculated?

We begin by calculating monthly revenue. For every month, we consider listings which are active for more than 75% of the times in the month, and are available to book for more than 50% of the times in the month (i.e. owner blocks < 50%)



In a given month, for every listing, we know when they got booked, and what price they got booked. We then calculate monthly revenue for these each of these listings. Once we have monthly revenue numbers for these listings, we calculate 25th, 50th, 75th percentile revenue numbers.



These percentile numbers we get for every month of the year.

The annual revenue number is sum of 50th percentile number for every month.

The lower range of annual revenue number is sum of 25th percentile number for every month.

The upper range of annual revenue number is sum of 75th percentile number for every month.



Does the revenue calculation includes any cleaning fee, taxes etc.

No, the revenue calculation is based on nightly rates. This means that we don't have any taxes, cleaning fee etc. except for the cases where host include their commission into nightly rates.



Other

How fast is the API call?

It depends on how much data we pull in. Typically for 350 listings, it takes about 6-8 seconds to estimate revenue.



What is the max number of bedroom category we can pass as a parameter in the API?

There is no capping on the bedroom category. We try to pull all available nearby listings data for a given bedroom category.



Will there ever be no revenue data for an address?

It is only possible if the given address (& nearby area) doesn't have any property listed on airbnb. In this case, we will send a zero response, and it will look like below -

A successful API response has the 200 code associated with it. The other codes are the error codes (400, 500, 403, 502, 404)
