Common extraction mistakes
Symptom	Typical cause
listingImagesCount shows 9 but UI is blank	Code is looking for listingImages (camel‑case) instead of listing_images.
Only the first photo shows	Using listing_images.image instead of mapping over the array.
Broken images (404)	Using the thumbnail URL for a full‑size <img> tag and the CDN blocks hot‑linking.
Nothing loads in dev but works in prod	Mixed‑content issue (HTTP image on an HTTPS site). Always use the HTTPS URL returned by the API.
Example extraction logic (JavaScript / TypeScript)
ts
interface ListingImage {
  id: number;
  image: string;      // full‑size
  thumbnail: string;  // small
  order: number;
  caption?: string;
}

function extractImageUrls(listing: any): string[] {
  // 1. Prefer the gallery; fall back to header images
  const gallery: ListingImage[] = listing.listing_images ?? [];
  const headers: ListingImage[] = listing.header_images ?? [];

  const combined: ListingImage[] =
    gallery.length > 0 ? gallery : headers;

  // 2. Sort by 'order' to keep the intended sequence
  combined.sort((a, b) => a.order - b.order);

  // 3. Return an array of URLs (use .thumbnail if that is what you need)
  return combined.map(img => img.image);
}

Quick checklist
Correct property name: listing_images (snake‑case).
Iterate over the array – each element is an object.
Pick the right field: image (large) or thumbnail (small).
Sort by order if you want the same sequence as the agent’s site.
Use HTTPS – the URLs returned by PropData are already HTTPS; don’t strip the protocol.
Handle empty arrays – some listings only have header_images (hero shots) and no listing_images.
If you still don’t see anything after following the above, log the first element of listing_images and confirm that:
js
console.log(listing.listing_images[0].image);
…prints a valid URL that you can open directly in the browser. If that URL works in the browser but not in your app, the problem is in the front‑end (CORS, CSP, or mixed‑content), not in the extraction step.
