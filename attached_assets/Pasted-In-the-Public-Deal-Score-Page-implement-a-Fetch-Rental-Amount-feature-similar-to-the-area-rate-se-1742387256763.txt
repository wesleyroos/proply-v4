In the Public Deal Score Page, implement a "Fetch Rental Amount" feature similar to the area rate service. This feature should call OpenAI’s API twice with independent prompts to estimate a property’s long-term rental value based on its address, size (sqm), number of bedrooms, and condition.

Backend Implementation (deal-advisor routes)
Create a new endpoint in deal-advisor.routes.ts:

Route: /api/deal-advisor/rental-amount
Method: POST
Accepts JSON payload:
json
Copy code
{
  "address": "27 Leeuwen St, Cape Town City Centre, 8001",
  "propertySize": 85,
  "bedrooms": 2,
  "condition": "Excellent"
}
Calls fetchRentalAmountService() to process the request.
Create a new service function in dealAdvisorService.ts:

Function: fetchRentalAmountService(address, propertySize, bedrooms, condition)
Process:
Calls OpenAI API twice with different expert perspectives:
First Prompt (Rental Market Expert):
"As a rental market expert in Cape Town, estimate the monthly rental price for a property at {address}. The property is {propertySize} sqm, has {bedrooms} bedrooms, and is in {condition} condition. Provide a single numeric value."
Second Prompt (Property Manager Perspective):
"As a property manager optimizing rental income, estimate the monthly rent for a {bedrooms}-bedroom, {propertySize} sqm property at {address} in {condition} condition. Focus on realistic, achievable rents."
Parses and validates responses:
Removes non-numeric values
Averages both estimates
Applies range validation (e.g., min R5,000, max R100,000)
Returns estimated monthly rent (ZAR).
Frontend Implementation (DealScorePublicPage.tsx)
Add a "Fetch Rental Amount" button next to the existing area rate fetch button.
Use handleFetchRentalAmount() function:
Calls /api/deal-advisor/rental-amount
Shows loading UI while waiting for response.
Fills the rental amount input field with the fetched result.
Error Handling & Debugging
Issue: 401 (Unauthorized) error on API call
Possible causes:

Invalid or missing API key/auth token – Ensure the frontend includes an authorization header with a valid API key.
CORS Policy – Check if the API server allows requests from the frontend domain.
Backend permissions – Verify that the deal-advisor API has the correct authentication setup.
