Below is a concise “cheat‑sheet” for working with PropData’s listings endpoints (staging environment) and extracting the fields you mentioned.
All URLs shown are for the staging gateway (https://staging.api‑gw.propdata.net).
The production gateway is usually identical but without the staging sub‑domain.
1. Authentication (if enabled)
http
GET /auth/token/   (or whatever your tenant uses)
Most PropData tenants use a simple Bearer token that is then sent on every request:
http
Authorization: Bearer <your‑token>
2. Key Residential‑listing endpoints
Purpose	Verb & Path	Notes
List all residential listings (paginated)	GET /listings/api/v1/residential/	Supports limit, offset, order_by.
Search residential listings	POST /listings/api/v1/residential/search/	Pass a JSON filters object (see below).
Fetch a single listing	GET /listings/api/v1/residential/{id}/	Returns the full record.
Commercial / holiday / project listings follow the same pattern, just replace residential with commercial, holiday, etc.
3. Detecting “new” listings
Option A – simple polling
http
GET /listings/api/v1/residential/?order_by=-created&limit=50
Keep the timestamp of the most recent created value you processed.
On the next poll, ignore anything older than that timestamp.
Option B – filtered search
http
POST /listings/api/v1/residential/search/
Content‑Type: application/json
Authorization: Bearer <token>

{
  "filters": {
    "created__gte": "2025-06-01T00:00:00Z",
    "status": ["active"]          // optional
  },
  "limit": 100
}
__gte, __lte, __contains, etc. follow Django‑filter syntax.
Typical filter keys (taken from the schema in the docs):
created, modified
status – "active", "sold", "archived"
branch, agent, suburb, property_type, price__lte, price__gte, etc.
4. Fields you asked for (from the example response)
JSON path	Meaning	Notes
street_number, street_name, suburb, city, region, country	Address	Combine as needed.
asking_price (or price)	Sales price	Currency is in price_currency.
property_type	Property type	e.g. "House", "Apartment".
bedrooms	Bedrooms	Integer.
bathrooms	Bathrooms	Integer.
garages / parkings	Parkings / Garages	Integer.
building_size / land_size	Size	Usually in m²; check size_unit.
status	Active / Sold / Archived	String enum.
created	Date listed by agent	ISO‑8601 timestamp.
modified	Last updated (useful for sync).	
valuation	Automated valuation (if enabled).	
latitude, longitude	Geo‑coords (for mapping / distance).	
tags, extras	Marketing tags or feature flags (pool, garden, etc.).	
images / header_images	Arrays of image URLs.	
5. Sample cURL snippets
List latest 20 active residential listings
bash
curl -H "Authorization: Bearer $TOKEN" \
     "https://staging.api-gw.propdata.net/listings/api/v1/residential/?status=active&order_by=-created&limit=20"
Fetch full detail for a single listing
bash
curl -H "Authorization: Bearer $TOKEN" \
     "https://staging.api-gw.propdata.net/listings/api/v1/residential/12345/"
Search for anything listed in the last 24 h
bash
curl -X POST \
     -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"filters":{"created__gte":"2025-06-02T00:00:00Z"}}' \
     "https://staging.api-gw.propdata.net/listings/api/v1/residential/search/"
6. Typical sync workflow
Initial back‑fill
Call GET /residential/?order_by=created&limit=100&offset=… until no more pages.
Store each listing in your DB keyed by id (primary key) and keep created / modified columns.
Incremental job (e.g. every 5 min)
Search endpoint with modified__gte=<last_run_timestamp> OR list endpoint ordered by -modified and stop once you hit a record you already have.
Upsert into your DB.
Frontend
Simple table view ordered by created DESC.
Show columns: Address, Asking Price, Property Type, Beds, Baths, Garages, Building Size, Status, Created.
Add filters for status / price range / suburb, etc.
Extra metrics for valuation
valuation – automated estimate.
distance (if you pass a lat/lon filter).
on_show_events – upcoming show dates.
tags – e.g. “sea‑view”, “fixer‑upper”.
extras – feature flags (pool, fireplace, security estate).
Quick JSON‑to‑DB mapping example (pseudo‑code)
python
for listing in response["results"]:
    db.upsert(
        id = listing["id"],
        address = f'{listing["street_number"]} {listing["street_name"]}, {listing["suburb"]}',
        price = Decimal(listing["asking_price"]),
        currency = listing["price_currency"],
        property_type = listing["property_type"],
        bedrooms = listing["bedrooms"],
        bathrooms = listing["bathrooms"],
        parkings = listing["garages"] or listing["parkings"],
        size = listing["building_size"] or listing["land_size"],
        status = listing["status"],
        date_listed = listing["created"],
        last_modified = listing["modified"],
        raw_json = listing  # keep full blob for any future field
    )
7. Common pitfalls
Pagination – default limit is often 20; always follow next in the response.
Time‑zones – created/modified are UTC; convert to local (en‑GB) if displaying.
Status vs. Availability – some clients use active/inactive; others rely on status plus a deleted flag.
Images – full URLs are already absolute; no extra host prefix needed.
That should give you everything you need to start polling PropData for new Sotheby’s listings, store them, and display them in your own frontend.
